### 6.2 导出逻辑

**JSON 导出（ZIP 包）：**
```
memos_export_20260529.zip
├── data.json
└── attachments/
    ├── photo.jpg
    └── photo_1.jpg  // 同名自动去重
```

data.json 结构：`{ version: 1, exportedAt: ISO时间, memos: [...] }`，memo 含 content/visibility/tags/pinned/createTime/attachments，attachment.path 为 `attachments/文件名`。

**Markdown 导出（ZIP 包）：**
每篇 memo 格式：
```
Tags: 标签1, 标签2
Date: 2026-05-29T12:00:00Z
Attachments: photo.jpg, doc.pdf

memo 正文内容

---
```

**关键实现点：**
- 附件 blob 用 `Promise.all` 并发 fetch，避免串行等待
- ZIP 直接用 `createZip(entries)`，不经过 exportService
- 文件名去连字符：`memos_export_20260529.zip`（macOS 兼容）
- 下载在所有 await 完成后触发
- 不设导入文件大小限制（自己用的实例，无需防 DDoS）

### 6.3 导入逻辑

4 步状态机：idle → preview → importing → done

**解析：** `parseImportFile` 支持 .zip 和 .json，解析后保存 `attachmentFiles` Map

**去重：** content + createTime 联合匹配，当前用户范围

**导入执行：**
1. 遍历每条新 memo
2. 上传附件：`attachmentServiceClient.createAttachment`，失败跳过
3. 创建 memo：`memoServiceClient.createMemo`，含 content/visibility/pinned/createTime(timestampFromDate)/attachments(引用)
4. 单条失败不影响后续，结果报告成功/跳过/失败数 + 错误列表

## 十、已知 Bug 及修复清单

以下是上一轮实施中踩过的坑，本轮直接按正确方式实施：

1. **`useMemos` 返回值**：数据在 `memoList.data?.memos`，不是 `memoList.data`
2. **`State` 导入**：从 `@/types/proto/api/v1/common_pb`，非 `memo_service_pb`
3. **ZoomableImage 初始不显示**：需 `max-h-full max-w-full object-contain`
4. **设置页位置**：import-export 必须在 SETTINGS_SECTIONS 数组中紧跟 webhook
5. **图片滑动跳张**：`key={previewItems[0]?.id}`，不是 `currentItem.id`
6. **视频 object-cover 裁切**：视频用 `object-contain`
7. **视频 poster 无效**：不要设 poster
8. **ZIP 文件名**：避免多个 `-` 连字符（macOS 兼容）
9. **下载时机**：所有 await 完成后再触发下载
10. **筛选按钮防抖**："清除"按钮始终渲染，用 `invisible` 控制可见性，不用条件渲染
11. **视频改动需两处一致**：AttachmentListView 和 PreviewImageDialog 中的 `<video>` 都用 `object-contain`、无 poster、`<video controls>`
12. **导出附件串行 fetch 性能差**：改用 `Promise.all` 并发 fetch 所有附件
13. **useMemos 无用户过滤**：`{ pageSize: 1000 }` 拉取全部用户 memo，应只用客户端 `.filter(m => m.creator === currentUser.name)`
14. **日期输入用原生 type="date"**：不用 showPicker() 方案（仅 Chromium 支持），直接用 `<input type="date">` 保证 Safari/Firefox/Chrome 全部兼容
15. **exportService 函数体死代码**：只保留 type/interface，删除 exportJsonZip/exportMarkdownZip 函数体
16. **PreviewImageDialog 旧 NavButton**：重写时删除内联 NavButton 组件定义
17. **Markdown 附件 MIME 类型**：`text/markdown`、`text/x-markdown` 都视为 md，同时 `text/plain` 中扩展名为 .md 的也归入 md（避免被归为文档类）
18. **MdPreviewDialog 认证**：fetch 读取 md 文件内容时需带 Bearer token（`getAccessToken()` + Authorization header），否则服务端返回 401
19. **导入时 pinned 字段丢失**：createMemo 时需传入 `pinned` 字段，否则置顶状态丢失

## 十三、实施修订记录

### 13.1 图片手势系统（§七）— 改用 fit 模型

放弃原任务书的 flexbox/absolute 两模式切换 + 手写双指锚点方案，改用成熟实现：

- 图片始终 `position: absolute`，`computeFit()` 计算拟合容器尺寸和位置 `{x, y, w, h}`
- `setZoom(z, cx, cy)` 保持容器坐标锚点固定：`ix = (cx - fitX - panX) / prevZoom`
- 全局 `window.addEventListener("mousemove/mouseup")` 处理拖拽（防止鼠标移到元素外丢失）
- `ResizeObserver` 监听容器变化，保持缩放状态下重新钳制 pan
- 滑动检测在 `touchend`，`lastTouchX - touchStartX` 判断方向
- 滚轮用 `Date.now() - lastNavRef > 500` 去抖
- 双击用 React `onDoubleClick`，带 `transform 0.3s ease-out` 过渡动画
- `lastTouchX/Y` 必须在 `onTouchStart` 中显式初始化为实际触摸坐标（newGestureState 默认为 0，否则轻触误触发滑动导航）

### 13.2 导入导出功能细化（§六）

**查重与增量更新：**
- 查重键：`content|createTime`（不含附件）
- 导入附件更多 → 调 `updateMemo` 只追加增量附件（不覆盖原有）

## 十七、Phase 7 — 滚轮缩放细调

> 实施日期：2026-05-31

### 17.1 问题背景

Apple HIG 修复后（Phase 6），wheel 事件仅 `ctrlKey/metaKey` 触发缩放。鼠标滚轮（不含 ctrlKey）掉到平移分支，1x 时被忽略 → **鼠标滚轮缩放失效**。

### 17.2 修复

| 问题 | 修复 | 说明 |
|------|------|------|
| 鼠标滚轮缩放失效 | `ady > adx && ady > 50` → 缩放 | 鼠标滚轮 deltaY ~100，触控板 ~5-20，50 阈值区分 |
| 滚轮步长过大 | `rate = ady > 50 ? 0.001 : 0.005` | 滚轮慢速细腻（~10%/刻度），捏合响应快（~5%/tick） |

### 17.3 当前缩放手势矩阵

| 输入 | 条件 | 因子 | 效果 |
|------|------|------|------|
| 捏合缩放 | ctrlKey/metaKey | 0.005 | 响应快 |
| 鼠标滚轮 | ady>adx && ady>50 | 0.001 | 细腻 |
| 触控板垂直 | ady>adx && ady≤50 | 不缩放（1x 忽略，放大后平移）|

### 17.4 Docker 镜像版本号修复

`release.yml` 未传 `build-args`，导致 Docker 镜像内版本显示 `dev`、commit 显示 `unknown`。现已通过 `docker/metadata-action` 输出的 `version` + `github.sha` 传入。
